# contexto/

Cache de execução do desenvolvimento do commandCenter.

Cada arquivo `.md` aqui documenta uma execução relevante de criação ou
adaptação do sistema (feature nova, mudança estrutural, integração,
migração de schema, decisão de arquitetura).

## Convenção

- Nome do arquivo: `YYYY-MM-DD-slug-curto.md`
- Conteúdo mínimo: o que foi feito, por quê, e quais arquivos/serviços
  foram tocados.
- Antes de rodar um build, os arquivos aqui devem ser relidos para
  reconstituir o estado acumulado do desenvolvimento.

Ver seção 18 do `CLAUDE.md` na raiz do repositório para a regra completa.
